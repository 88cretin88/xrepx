OpenBSD Зеркала

Загрузка через HTTP/FTP

OpenBSD можно скачать через HTTP(s) или FTP. В принципе, для загрузки вам нужна лишь совсем маленькая часть того, что есть на серверах. Остальное (устанавливаемые компоненты) может быть установленно из различных мест во время установки системы, в том числе и скаченно из интернета. Загляние в руководство по установке, если вам нужны подробности, касательно этого процесса.

Выберите ближайшее к вам зеркало из списка HTTP(s), FTP и rsync зеркал.

Обратите внимание на структуру каталогов зеркал и информацию по поводу планируемых зеркал.

Структура каталогов:

Структура каталогов каждого зеркала выглядит следующим образом:
  • /pub/OpenBSD/
    Абсолютный PATH для зеркал, где должна быть доступна коллекция FTP OpenBSD. Это может быть ссылкой, если это необходимо.

  • /pub/OpenBSD/6.6/
    Где можно найти последний релиз.
    • alpha/
    • amd64/
    • arm64/
    • armv7/
    • hppa/
    • i386/
    • landisk/
    • loongson/
    • luna88k/
    • macppc/
    • octeon/
    • packages/
      Бинарные пакеты (скомпилированные порты) для установки в -release или -stable системах.
    • sgi/
    • sparc64/
    • src.tar.gz
      Userland исходники, которые находятся в /usr/src.
    • sys.tar.gz
      Kernel исходники, которые находятся в /usr/src/sys.
    • ports.tar.gz
      Дерево портов, которое находится в /usr/ports.
    • xenocara.tar.gz
      Исходники X window system, которые находятся в /usr/xenocara.

  • /pub/OpenBSD/snapshots/
    Для наших основных архитектур мы часто собираем релизы и бинарные пакеты из нашего репозитория. Здесь мы размещаем эти тестовые релизы, которые зовутся снапшотами.

  • /pub/OpenBSD/tools/
    Другое различные маленькие вещи.

  • /pub/OpenBSD/6.5/
    Такая же структура каталогов как и для релиза 6.6, описанного выше.

Информация для планируемых/поддерживаемых зеркал

Чтобы попасть в список официальных зеркал (на этой странице), мы требуем, чтобы зеркала были быстрыми, современными, работоспособными и не имели никаких проблем с сетевым подключением.

Зеркала должны предоставлять:

  • ОС и пакеты для всех архитектур
  • бинарники по крайней мере для двух последних релизов
  • свежие (up-to-date) снапшоты (не старше, чем неделя)
  • каталоги для всех проектов
    (включая OpenSSH/, OpenNTPD/, OpenBGPD/, LibreSSL/, patches/, syspatch/ and tools/)

В дополнее к вышесказанному, зеркала должны использовать second-level зеркала в качестве upstream.

Начиная с 6.5 минимальное требуемое место на диске составляет приблизительно 700 Гб. Однако для уменьшения проблем при обновлении пакетов для снапшотов рекомендуется использовать rsync параметры --delete-delay --delay-updates, которые будут использовать лишь дополнительное место. --fuzzy также может быть полезен для уменьшения объема данных, передаваемых при обновлении пакетов для снапшотов. В зависимости от свободного места, зеркала могут предоставлять больше. Например, репозитории для старых релизов, репозиторий с исходниками и т.д.

Предлагаемая конфигурация httpd.conf(5) для httpd(8), которая подходит для www- и ftp-зеркал, здесь.

Операторы зеркал, которые заинтересованы в том, чтобы их машина была добавлена в список на этой странице, должны сообщить об этом в рассылку mirrors-discuss@openbsd.org, где нужно сообщить подробную информацию о своем зеркале, включая характеристики сетевого подключения, географическое местоположение и общую информацию. Мы предложим подходящее зеркало второго уровня (second-level) для синхронизации; не зеркальте с ftp.openbsd.org.

Все операторы зеркал должны быть подписанны на почтовую рассылку mirrors-announce (трафик низкий).
